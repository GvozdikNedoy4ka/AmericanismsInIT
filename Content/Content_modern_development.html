Термины Определения
Containerization Методология виртуализации, позволяющая упаковывать приложения и их зависимости в легко переносимые контейнеры для более эффективного развертывания и управления.
Microservices Архитектурный подход к разработке программного обеспечения, в котором большие приложения разбиваются на небольшие, автономные сервисы, взаимодействующие между собой через API.
Monolithic Это подход к разработке программного обеспечения, при котором весь функционал приложения разворачивается в единственном исполняемом экземпляре. В такой архитектуре весь код, компоненты и функции приложения обычно разворачиваются вместе как единое целое. То есть, весь код работает в одном процессе и обычно разворачивается на одном сервере.
Serverless Computing Модель облачных вычислений, в которой облачный провайдер управляет выполнением кода, а пользователь платит только за ресурсы, фактически используемые приложением, без необходимости управления серверами.
Continuous Integration (CI) Практика разработки программного обеспечения, при которой изменения в коде регулярно интегрируются в общий репозиторий и автоматически проверяются на наличие ошибок.
Continuous Deployment (CD) Расширение практики CI, при котором каждое успешное изменение автоматически развертывается в производственную среду без необходимости ручного вмешательства.
Virtualization Технология, позволяющая создавать виртуальные версии аппаратного и программного обеспечения, что позволяет эффективнее использовать ресурсы компьютера и упрощает управление инфраструктурой.
API Economy Экономическая модель, в которой компании предоставляют открытые API для своих продуктов и сервисов, что позволяет сторонним разработчикам создавать приложения и интеграции на их основе.
